4.5 HW 구성
4.5.1 외부 인터페이스
  1. 1) 통신 모듈 제어, 단말 디버깅, 펌웨어 업그레이드를 위해 PC 와 연결 가능한 인터페이스 제공
    1. ① CM Port 제공(UART 또는 USB) : 단말 제어(AT CMD)가 가능한 CM툴을 GUI 형태로 제공
    2. ② DM Port 제공(RS-232C, UART, USB 또는 LAN 포트) : kt Air Message 분석 툴인 KDM 을 위한 포트 제공, KDM 연동 불가능한 경우 별도 분석 툴 제공
  2. 2) DM/CM 필수 요구 사항
    1. ① DM/CM/Console 등 외부에 노출된 Port 접근 시 권한 설정 기능 구현
      • 상용 단말에서 보안상 DM Port를 비활성화(Disable) 상태로 설정 되어야 함
      • DM Port 활성화(Enable) 시 접근권한 (예 : OTP 방식 등)을 설정 하여야 함
      • DM Port 활성화 또는 비활성화 설정 시 단말이 리부팅 되지 않고 기능이 적용되어야 함
      • CM/Console 등 단말 설정을 하기 위한 Port 접근 시 접근권한 (예 : OTP 방식 등)을 설정해야 함
      • 기능검증 진행 시 DM Port 활성화가 가능하여야 하므로, 활성화 방법은 별도 전달
  3. 3) DM/CM 논리적 구성(예)

4.6 품질관련 주요 기능 구현

하기의 필수로 구현되어야 할 항목은 상용 단말에도 동일하게 적용되어야 함

4.6.1 단말 기능
구분 기능항목
상용화 이후 원격 지원 기능
  1. (1) 제조사 FOTA 적용
    • 제조사 서버를 통한 모뎀/모듈 FOTA 적용
    • 제조사 서버를 통한 단말 FOTA 적용
    • 제조사 서버를 통한 단말/모뎀 원격 Reset 기능 적용
    • 제조사 FOTA 적용 시 kt 승인 F/W만 현장적용 가능
    • kt 미승인 F/W적용으로 발생하는 모든 문제의 후속처리는 제조사 비용으로 처리
네트웍 망영향성 검토 및 부하 경감
  1. (1) 단말 서비스 시나리오별 데이터 접속 기능 확인
    1. 1) Attach/Detach 기능 확인
    2. 2) 주기보고 및 이벤트보고 기능 확인
      1. ① 주기보고/이벤트보고 시간에 맞게 데이터 전송 가능
      2. ② 데이터 중복/누락이 발생하지 않아야 함
      3. ③ 주기보고 시간이 5분 이상 일 경우 주기보고 시간 변경이 가능 (단, 펌웨어 업데이트 없이 주기보고 변경이 가능하여야 함)
    3. 3) Preservation 기능 확인 (망 자원 해제 안 하는 단말)
  2. (2) 망/데이터 접속 실패 시 재접속 시나리오 확인
    1. 1) 데이터 무한 재전송 시도 제한 기능
      • ① 통신망 접속 실패 시 무한 재전송 금지
      • ② 서비스 플랫폼 접속 실패 시 무한 재전송 금지
      • ③ 서비스 플랫폼 에러코드 수신 시 무한 재전송 금지
      • ④ 통신망 에러코드 수신 시 무한 재전송 금지
    2. 2) 데이터 전송 실패 시 전송 데이터 신뢰성 확보 방안
      • ① 본보고 포함 최대 3회까지 재 전송
      • ② 재전송 간격 및 횟수는 망 영향성 검토 시 kt 네트워크 부서와 협의
  3. (3) 일괄 재부팅 불가
    1. 1) 특정 시간대에 모든 단말이 일괄적으로 재부팅하는 기능 적용 금지
    2. 2) 단말별 분산 재부팅 알고리즘을 적용 (예 : IMEI 별 재부팅 시간 적용 등)
  4. (4) 미 전송 패킷 발생 시 처리 기능 구현
    1. 1) 단말 서비스 시나리오가 있을 경우에는 단말이 서비스 시나리오를 준수
    2. 2) 정의되지 않았을 경우 단말에 저장하지 않아야 하며, 통신이 복구된 이후에는 일괄 전송이 되지 않아야 함
  5. (5) 네트워크부문 망영향성 검토
    • 별도 양식 작성을 통한 kt 내부 검토
품질관련 주요 기능
  1. (1) 단말 비정상 복구 기능 구현
    1. 1) 단말 Health Check 기능 구현
      1. ① 단말이 정상 동작 중인지 확인을 위한 Watch dog 기능 제공
      2. ② Watch Dog 기능이 구동 정지된 경우 재실행 할 수 있는 기능 제공
    2. 2) ICMP 기능으로 단말 상태 기능 구현
      1. ① ICMP Check를 통한 단말이 정상 동작 중인지 확인할 수 있는 기능 제공
      2. ② ICMP Check는 외부 서버와 교번으로 패킷이 정상 전송되는지 확인할 수 있어야 함 (Ex. A서버 → B서버 → C서버 → D서버 IP로 순차적 확인)
        • 순차 확인 시간은 30초, 1분 중에서 제조사 단말에 따른 선택 적용
        • 순차적인 ICMP Check가 5분간 연속 FAIL 발생한 경우 단말 복구 기능 제공
    3. 3) 무선망 세션 삭제 시 복구 기능 구현
      • 네트워크 코어 망 세션이 삭제(망 작업 또는 망 장애 상황 등)되는 경우 단말은 정상적으로 데이터 접속을 복구할 수 있는 기능 제공
  2. (2) 원격 APN 변경 기능 제공
    • 원격(서비스 플랫폼, SMS, kt DMS 등)으로 APN 변경 기능 구현
  3. (3) APN 복구 기능
    • 원격변경(kt DMS, 관제, SMS(ex. PW-OPT방식) 등) 또는 수동(CM Tool, AT CMD, WEB CM, 엔지니어모드 등)으로 APN 변경 시 오류가 발생하였을 경우 이전에 정상적으로 사용하던 APN으로 복구
